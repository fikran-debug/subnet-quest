IP Address & Struktur Dasar
IP Address adalah alamat unik setiap perangkat dalam jaringan — seperti alamat rumah, tapi untuk komputer dan router.

🔢 Struktur IPv4

IPv4 terdiri dari 32 bit yang dibagi menjadi 4 kelompok (oktet), masing-masing 8 bit.

Nilai setiap oktet: 0 sampai 255 (karena 8 bit = 2⁸ = 256 kemungkinan nilai).

🏷️ Kelas IP Address

IPv4 dibagi menjadi kelas berdasarkan nilai oktet pertama:

KelasRange Oktet 1Default MaskPenggunaan
A1 – 126/8Jaringan sangat besar
B128 – 191/16Jaringan menengah
C192 – 223/24Jaringan kecil (LAN)
D224 – 239—Multicast
💡 Di Cisco Packet Tracer: Kamu akan sering bekerja dengan IP Kelas C seperti 192.168.x.x karena itu yang paling umum digunakan di jaringan LAN sekolah dan kantor.

🔒 IP Privat vs Publik

Tidak semua IP bisa diakses dari internet. Ada IP khusus untuk jaringan internal (privat):

Kelas A: 10.0.0.0 – 10.255.255.255
Kelas B: 172.16.0.0 – 172.31.255.255
Kelas C: 192.168.0.0 – 192.168.255.255

IP ini bebas digunakan di jaringan lokal dan tidak bisa diakses langsung dari internet.

🎭 Tab 2 dari 4
Subnet Mask & CIDR
Subnet mask adalah "penanda" yang memisahkan bagian Network ID dan Host ID dalam sebuah IP Address.

✂️ Network ID vs Host ID

Bayangkan IP seperti alamat surat: Nama Jalan = Network ID, Nomor Rumah = Host ID.

IP: 192.168.1. 50
Mask: 255.255.255. 0
─────────────────────
Net: 192.168.1. [Network]
Host: ───────────── 50 [Host]

📏 Notasi CIDR

CIDR (Classless Inter-Domain Routing) adalah cara ringkas menulis subnet mask menggunakan angka setelah tanda "/".

CIDRSubnet MaskBit HostMax Host
/24255.255.255.08254
/25255.255.255.1287126
/26255.255.255.192662
/27255.255.255.224530
/28255.255.255.240414
/29255.255.255.24836
/30255.255.255.25222
⚠️ Ingat! Angka di CIDR (/24, /26, dst) menunjukkan jumlah bit yang "dipakai" untuk Network. Sisanya digunakan untuk Host. Jadi /24 = 24 bit network, 8 bit host.

🏠 3 Alamat Penting Setiap Subnet

AlamatPosisiKeterangan
NetworkPertamaIdentitas jaringan, tidak bisa dipakai host
Host RangeTengahAlamat yang bisa dipakai perangkat
BroadcastTerakhirKirim ke semua host, tidak bisa dipakai host
Contoh /24:

Host 1 : 192.168.1.1

🔢 Tab 3 dari 4
Rumus & Cara Hitung
Ini adalah inti dari subnetting — memahami rumus ini akan memudahkan kamu menjawab soal dan konfigurasi di Packet Tracer.

📐 Rumus Utama

Bit Host (n) = 32 − CIDR

Jumlah Host = 2ⁿ − 2
Jumlah Subnet = 2ˢ (s = bit dipinjam)
Block Size = 256 − nilai oktet mask

Kenapa dikurang 2? Karena 1 untuk Network Address dan 1 untuk Broadcast Address.

🧮 Langkah Menghitung Subnet

Contoh: Diketahui IP 192.168.1.0/26

Langkah 1: Bit host = 32 − 26 = 6
Langkah 2: Jumlah host = 2⁶ − 2 = 62
Langkah 3: Block size = 256 − 192 = 64
Langkah 4: Subnet = .0, .64, .128, .192

Subnet 1: 192.168.1.0 − 192.168.1.63
→ Network : 192.168.1.0
→ Host 1 : 192.168.1.1
→ Host End: 192.168.1.62
→ Broadcast: 192.168.1.63

⚡ Trik Cepat Block Size

Block size = jarak antar subnet. Cara cepat: 256 − nilai oktet terakhir subnet mask.

CIDROktet Terakhir MaskBlock Size
/25128256−128 = 128
/26192256−192 = 64
/27224256−224 = 32
/28240256−240 = 16
/29248256−248 = 8
/30252256−252 = 4
💡 Tips saat ujian/game: Hafal block size! Dari situ kamu langsung bisa hitung Network Address, Broadcast, dan Host Range tanpa perlu konversi biner.

Contoh Kasus Lengkap
Pelajari contoh berikut sebelum masuk ke game. Pastikan kamu paham alur perhitungannya!

📌 Kasus 1: /27

Diketahui: 192.168.10.0/27 — Tentukan semua subnet!

Bit host = 32−27 = 5
Host/subnet = 2⁵−2 = 30
Block size = 256−224 = 32

Subnet 1: .0 → .31 (Host: .1–.30)
Subnet 2: .32 → .63 (Host: .33–.62)
Subnet 3: .64 → .95 (Host: .65–.94)
Subnet 4: .96 → .127 (Host: .97–.126)
... dst (total 8 subnet)

📌 Kasus 2: Konfigurasi di Packet Tracer

Skenario: Kamu perlu menghubungkan 50 PC dalam 1 jaringan. CIDR yang cocok?

Butuh min 50 host
/26 → 62 host ✓ (cukup!)
/27 → 30 host ✗ (kurang)

Pilih /26 → 255.255.255.192
Config di Packet Tracer:
IP: 192.168.1.1
SM: 255.255.255.192
